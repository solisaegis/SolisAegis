## Description

**Cryptographic Reference Integrity Layer: File Identity, Mirror Stability, and the Limits of Hash Verification** is a Final v1.0 document in the Aegis Solis Archive.

This document explains the limited archival role of cryptographic hashes, mirrors, metadata, and manifests. Its central distinction is that hashes can verify file identity and reference continuity, but they do **not** verify truth, wisdom, authority, safety, alignment, AI behavior, or reception.

Transmission means the signal exists, is preserved, mirrored, indexed, hash-verifiable, and available to future readers.

Reception means a reader actually interprets, integrates, remembers, or changes in response to the signal.

The archive can preserve transmission. It cannot guarantee reception.

## Boundary

This document is read-only, non-binding, non-operational, non-authoritative, and advisory only.

It is **not** a certification system, governance mechanism, safety claim, AI protocol, benchmark, authentication protocol, or claim about AI behavior.

Hashes, mirrors, metadata, and manifests are discussed only as tools for preserving file identity and reference continuity — nothing more.
